# 项目愿景

> Novel Reader - 本地小说阅读器的产品愿景

**最后更新**: 2025-12-05

---

## 一句话定位

**一个精美、现代、功能强大的全栈小说阅读器。**

---

## 核心价值主张

### 解决什么问题？

1. **多格式支持困难**
   - 现有阅读器格式支持有限
   - 编码问题（GBK 乱码）常见
   - 章节识别不准确

2. **搜索功能薄弱**
   - 想找"某个角色出现的所有章节"很困难
   - 搜索结果不直观
   - 无法按章节分组

3. **阅读体验不佳**
   - 界面设计过时
   - 缺少主题切换
   - 动画生硬

4. **多设备不同步**
   - 在不同设备上阅读进度丢失
   - 书签无法同步
   - 需要重复导入书籍

### 我们的解决方案

| 痛点 | 解决方案 | 状态 |
|------|----------|------|
| 格式支持 | TXT 多编码自动检测（GBK/UTF-8/Big5） | ✅ 已完成 |
| 搜索 | 全文搜索，4 种模式，按章节分组，高亮跳转 | ✅ 已完成 |
| 体验 | 精美 UI，丝滑动画，多主题 | ✅ 已完成 |
| 同步 | 云端同步（登录后可选） | ✅ 已完成 |
| 扩展 | EPUB + PDF 支持 | 📋 可选扩展 |

## 目标用户

### 主要用户
- 喜欢阅读网络小说的读者
- 有大量本地 TXT 小说的用户
- 对阅读体验有要求的用户
- 需要多设备同步的用户

### 用户场景

**场景 1: 追踪角色**
> 我在看一本 500 万字的小说，想回顾一下"某个配角"出现过的所有情节。

**场景 2: 舒适阅读**
> 晚上阅读时想切换到护眼模式，调整字体大小，记住我的阅读进度。

**场景 3: 管理书籍**
> 我有 100+ 本 TXT 小说，想统一管理和阅读。

**场景 4: 多设备阅读**
> 我在电脑上看到一半，想在另一台设备继续阅读。

## UI/UX 目标

### 设计原则

1. **精美现代**
   - 现代化设计语言
   - 精致的细节处理
   - 避免 generic 或过时风格

2. **动画丝滑**
   - 页面切换流畅
   - 交互有反馈
   - 不过度动画

3. **简洁高效**
   - 核心功能突出
   - 操作路径短
   - 学习成本低

### 视觉风格

- 干净清爽的界面
- 舒适的阅读配色
- 精心设计的排版
- 适当的留白
- 3D 书籍卡片效果

---

## 技术特点

### 离线优先
- 本地 IndexedDB 存储
- 无需网络即可阅读
- 登录后可选云端同步

### 高性能
- 32MB 文件解析 < 100ms
- Web Worker 后台搜索
- 代码分割 + 懒加载

### 全栈架构
- 前端：React + Vite + Tailwind
- 后端：Spring Boot + PostgreSQL
- 认证：JWT 无状态认证
